# v0.2.0-beta Release Notes

`v0.2.0-beta` is the first public beta release of `DevQuizAPI`.

## Highlights

- Public onboarding is significantly improved with clearer README guidance, import documentation, and troubleshooting notes
- Quiz import, ZIP package handling, and quiz image rendering are working end to end
- Admin and quiz pages have been polished for a more consistent browser experience
- Integration tests cover the core authentication, quiz, and import flows
- GitHub Actions CI now validates builds and tests automatically

## Included In This Release

- automatic development database migration on startup
- seeded development admin account
- seeded fake sample quiz data for fresh development databases
- public landing page at `/`
- Swagger moved to `/swapi.html`
- improved ZIP upload and extracted-CSV import workflow
- uploaded file listing aligned with the real storage path
- quiz image import and display support
- `/health` endpoint for deployment checks
- production configuration validation on startup
- deployment guide, changelog, release checklist, import documentation, and FAQ
- public sample quiz package included in the repository

## Beta Readiness Improvements

- GitHub-friendly README links and public-facing documentation cleanup
- clearer admin success and error messaging
- consistent light-theme styling across landing, admin, and quiz pages
- admin-only registration endpoint hardening
- FAQ and troubleshooting guidance for authentication, imports, and production startup

## Recommended Audience

This release is appropriate for:

- local development
- public evaluation
- technical preview users
- early adopters testing quiz import and quiz-taking workflows

## Known Beta Limitations

- deployment and release publication are still manual workflows
- production hosting is documented but not yet automated
- broader end-user feedback and bug reports are still expected before a stable release

## Upgrade Notes

- if you imported quiz packages before image support was completed, re-import them so image records are written correctly
- if you are using Swagger for protected endpoints, authorize with `Bearer <token>` after login

## Next Focus Areas

- feedback-driven bug fixing
- additional usability polish
- further production hardening
- stabilization toward a future `v1.0.0`
